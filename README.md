Приложение для подсчета финансов.

Фреймворк для работы с api: FastAPI
Фреймворк для работы с БД: SQLAlchemy


запуск сервера через команду в терминале:




uvicorn main:app --reload

main - путь к нашему файлу
app - инстанс класса
--reload - параметр, который перезапустит сервер
закрыть сервер Ctrl+C
документация {local_host}/docs
Например: http://127.0.0.1:8000/docs

Если сервер не обновляется, закрыть его и в терминале отправить:
tasklist | findstr python
взять номер задачи и следом отправить:
taskkill /PID <номер_PID> /F например taskkill /PID 10960 /F
запустить сервер как обычно



на 46:37 закончена начальная архитектура. Она состоит из 3х слоев: роутер(v1), сервис(service) и репозиторий(repository)
Слои:
- Роутер отвечает только за то, чтобы показывать, какие есть endpoints и их версии (если будем поддерживать версионирование)
- Сервиса реализует всю бизнес-логику приложения (там методы работы эндпоинтов, в которых вызываются более мелкие методы из слоя репозиторий)
- Репозиторий поддерживает взаимодействие с данными. Работа с данными только в этом слое. Слои сервис и роутер не могут взаимодействовать с данными.
Все взаимодействия с данными должны происходить только в слое Репозитория.
Зависимость слоев:
Слой роутера зависит от слоя Сервиса. Слой сервиса зависит от слоя репозитория. 
Схема: Репозиторий -> Сервис -> Роутер.
Теория архитектуры такая: Зависимость слоев прямая. Один слой зависит от другого. Нижележащие слои не знают о верхнестоящих.
